# 🔐 ISO 27001 Risk Assessment (Network-Focused)

## 📌 Project Overview
This project presents a comprehensive risk assessment conducted in alignment with ISO/IEC 27001 standards for a simulated mid-sized IT organization, Endpoint Technologies PLC.

The assessment focuses on identifying, analyzing, and treating information security risks across network infrastructure, systems, endpoints, and data assets.

---

## 🎯 Objectives
- Identify critical information assets within the organization
- Analyze potential threats and vulnerabilities
- Evaluate risks using a structured likelihood-impact model
- Recommend mitigation strategies aligned with ISO 27001 Annex A controls
- Strengthen overall security posture focusing on network security and access control

---

## 🏢 Organization Profile
**Name:** Endpoint Technologies PLC  
**Industry:** Software Development & IT Services  
**Size:** 50–100 employees  

### Core Operations:
- Web and mobile application development  
- API-based enterprise services  
- Internal SaaS solutions  

### Key Information Assets:
- Client project data  
- Source code repositories  
- Employee credentials  
- Internal communication systems  
- Financial records

## 🧠 Methodology

### 🔹 Risk Identification
Assets, threats, and vulnerabilities were identified across:
- Network infrastructure  
- Servers and applications  
- Endpoints  
- Data assets  

---

### 🔹 Risk Analysis
Risk scoring was calculated using:

> **Risk Score = Likelihood × Impact**

#### Likelihood Scale:
| Value | Meaning |
|------|--------|
| 1 | Rare |
| 2 | Unlikely |
| 3 | Possible |
| 4 | Likely |
| 5 | Almost Certain |

#### Impact Scale:
| Value | Meaning |
|------|--------|
| 1 | Negligible |
| 2 | Minor |
| 3 | Moderate |
| 4 | Major |
| 5 | Critical |

---

### 🔹 Risk Levels
| Score | Level |
|------|------|
| 1–5 | Low |
| 6–12 | Medium |
| 15–25 | High |

## 🔍 Key Risk Areas Identified

### 🔹 Network Security
- Lack of VLAN segmentation (lateral movement risk)
- Weak router/switch configurations
- Over-permissive firewall rules

---

### 🔹 Application Security
- SQL Injection and XSS vulnerabilities
- Outdated software components
- Lack of Web Application Firewall (WAF)

---

### 🔹 Data Security
- Lack of encryption at rest
- Weak credential management
- Exposure of sensitive data

---

### 🔹 Endpoint Security
- Malware infections
- Phishing attacks
- Lack of endpoint protection

---

### 🔹 Business Continuity
- Single ISP dependency
- Inadequate backup isolation

---

## 🛡️ Security Improvements Implemented

- Multi-Factor Authentication (MFA)
- Role-Based Access Control (RBAC)
- VLAN-based network segmentation
- Web Application Firewall (WAF) deployment
- Encryption for sensitive data
- Air-gapped backup strategy
- Email security (SPF, DKIM, DMARC)

---

## 📉 Residual Risk

After implementing controls:
- Most risks reduced to **Low or Medium**
- Some risks remain:
  - DDoS attacks  
  - ISP dependency  
  - Human-related threats (phishing)
